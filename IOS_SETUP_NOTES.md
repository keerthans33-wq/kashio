# iOS Setup Notes (Capacitor)

## What was added

Capacitor was added to `apps/web` to wrap the existing Next.js web app in a native iOS shell.
No existing web app code was changed. No Supabase schema, Stripe, or paywall logic was touched.

### New files and directories

| Path | What it is |
|------|------------|
| `apps/web/capacitor.config.ts` | Capacitor configuration (app ID, name, server URL) |
| `apps/web/ios/` | Native Xcode project generated by Capacitor |
| `apps/web/.gitignore` | Updated to exclude iOS build artefacts (Pods, DerivedData) |

### New npm packages (apps/web)

| Package | Role |
|---------|------|
| `@capacitor/core` | Core Capacitor runtime |
| `@capacitor/ios` | iOS platform bridge |
| `@capacitor/cli` | CLI tooling (devDependency) |

---

## How it works

The iOS shell uses Capacitor's `server.url` mode. Instead of bundling a static export of
the Next.js app, the iOS WebView loads the live deployed web app at:

```
https://app.kashio.com.au
```

This means:
- All server-side features (API routes, auth, Prisma, Stripe) keep working exactly as on web.
- No static export is required — `next build` is unchanged.
- The iOS app is a thin native wrapper around the live web app.

## Prerequisites before opening in Xcode

1. **Xcode** installed (Mac App Store, version 15 or later recommended)
2. **CocoaPods** installed:
   ```
   sudo gem install cocoapods
   ```
3. **Node.js** available in your PATH

---

## Commands to open in Xcode

Run these from `apps/web/`:

```bash
# 1. Install CocoaPods dependencies (first time only, or after cap sync)
cd ios/App && pod install && cd ../..

# 2. Open the workspace in Xcode
npx cap open ios
```

Xcode will open `apps/web/ios/App/App.xcworkspace`. Select a simulator or a connected
iPhone, then press the Run button (▶).

---

## Updating the iOS project after config changes

If you change `capacitor.config.ts` or add Capacitor plugins, run:

```bash
# From apps/web/
npx cap sync ios
```

This copies config updates into the native project.

---

## Switching from live URL to a local build (future)

When you are ready to ship offline-capable features or need a bundled build:

1. Add `output: 'export'` to `next.config.ts` (note: this disables API routes and server
   components — plan the migration carefully).
2. Run `next build` — output goes to `apps/web/out/`.
3. Remove the `server.url` block from `capacitor.config.ts`.
4. Run `npx cap sync ios`.
